# Nebius AI Performance Engineering Notebook Workspace

This repository is a notebook-first workspace for the Nebius AI Performance Engineering course. It contains the shared course notebooks, supporting visual assets, and a tiny Python package used only for a sanity check.

Most implementation work lives in Jupyter notebooks under `src/`. The notebooks in this project cover optimization fundamentals, PyTorch training loops, text-classification work, and shared practice material.

HW4 has been split into a standalone sibling VS Code Python project at `../hw4`. That project now owns the HW4 CIFAR-10 notebooks, character-level RNN language-model notebook, NumPy forward/backward-pass bonus notebooks, HW4 data files, and HW4-specific architecture notes.

## Current Contents

### Core Homework Notebooks

- [`src/hw1/HW_1_sub.ipynb`](src/hw1/HW_1_sub.ipynb): earlier homework submission covering text-classification and optimization tasks.
- [`src/hw2/HW2_Gradient_descent_&_Pytorch.ipynb`](src/hw2/HW2_Gradient_descent_&_Pytorch.ipynb): gradient descent, PyTorch autograd, optimizer behavior, weather classification, and softmax stability work.

### Extracted HW4 Project

- [`../hw4`](../hw4): standalone HW4 project with its own `README.md`, `CHANGELOG.md`, `pyproject.toml`, `uv.lock`, `.venv`, VS Code settings, local `data/`, and new git repository.

- [`src/hw2/pytorch_optimization_report.ipynb`](src/hw2/pytorch_optimization_report.ipynb): compact optimization report notebook.

- [`src/basics-test.ipynb`](src/basics-test.ipynb): small local notebook checks and exploratory experiments.
- [`docs/LLM_Optimization_Tutorial_v3.pdf`](docs/LLM_Optimization_Tutorial_v3.pdf): supporting optimization tutorial material.
- [`docs/gradient_descent.gif`](docs/gradient_descent.gif), [`docs/gradient_descent_lr_0.1.gif`](docs/gradient_descent_lr_0.1.gif), and [`docs/rosenbrock_shared_compass_fixed_1.gif`](docs/rosenbrock_shared_compass_fixed_1.gif): optimization visualization assets.

Local downloaded datasets and runtime artifacts should stay out of git unless intentionally promoted to source artifacts. The HW4-only CIFAR-10, dinosaur-name, and MNIST CSV data files were moved to `../hw4/data`; this repository still keeps the HW2 weather data cache under `src/data/`.

## Environment

Typical local setup:

```bash
uv sync --group dev
uv run pytest
uv run python -m workspace
```

To start Jupyter from the managed uv environment:

```bash
uv run jupyter lab
```

## Working Model

Use this repository as an interactive notebook workspace:

1. Open the relevant notebook under `src/hw1/`, `src/hw2/`, or the shared practice notebooks under `src/`.
2. Execute cells sequentially to reproduce setup, training, plots, and conclusions.
3. Use the sibling `../hw4` project for the extracted HW4 notebooks and longer CIFAR-10/RNN runs.
4. Keep downloaded datasets and generated runtime files out of git unless they are intentionally promoted to source artifacts.
